Startup Arguments

Run the application with your username and database selection

Overview

MTM Inventory Application supports startup arguments so you can choose which database to use (Production or Test) and optionally provide a username and database credentials when launching the app.

These options are helpful for QA testing, training environments, shared terminals, or kiosk setups.

Available Options

  • -db=prod → Use the Production database (mtm_wip_application_winforms)
  • -db=test → Use the Test database (mtm_wip_application_winforms_test)
  • -db=<customName> → Use a specific database name (advanced)
  • -user=<YourName> → Sets the in-app username shown in logs and settings
  • -dbuser=<dbUser> → Database username (optional; defaults to root)
  • -dbpassword=<dbPass> or -dbpass=<dbPass> → Database password
  • -password=<dbPass> → Also treated as the database password (shortcut)
Arguments can also be written with a forward slash prefix (e.g., /db=prod), which works the same.

Examples

Use Production DB with your app username

"C:\\Path\\To\\MTM_WIP_Application_Winforms.exe" -db=prod -user=JDOE

Use Test DB and provide database credentials

"C:\\Path\\To\\MTM_WIP_Application_Winforms.exe" -db=test -dbuser=root -dbpassword=root

Use a custom database name

"C:\\Path\\To\\MTM_WIP_Application_Winforms.exe" -db=mtm_wip_application_winforms_training

Create a Desktop Shortcut (Recommended)

  1. Right-click the desktop and choose New → Shortcut.
  2. For the location, click Browse… and select the app EXE, then add your arguments.
    "C:\\Apps\\MTM\\MTM_WIP_Application_Winforms.exe" -db=prod -user=JDOE
  3. Click Next, name it (e.g., MTM WIP - Production), then click Finish.
  4. Optional: Right-click the new shortcut → Properties. In Target, confirm the full command including arguments is present.
Create separate shortcuts for Production and Test to avoid mistakes.

How to Verify It Worked

  • Open the app and perform a simple database action (e.g., load Inventory).
  • If you used -db=test, the data should reflect the Test environment.
  • Check application logs if enabled (View → Application Logs) to see the effective database name.

Security Notes

If you put a password in a shortcut target field, anyone with access to your computer can see it.
  • Only use -dbpassword on secured machines or test environments.
  • For production, prefer secured configurations managed by IT when available.
  • Remove the password from the shortcut if you no longer need it.

Troubleshooting

  • App won't start or shows DB error: Double-check the database name and network connectivity.
  • Still connecting to the wrong DB: Reopen the shortcut and confirm the Target has the correct -db= value.
  • Username didn’t update: Confirm the -user= value is present and there are no extra spaces.
