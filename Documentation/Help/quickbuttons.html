üöÄ Using QuickButtons

Location and Layout

QuickButtons appear in the right panel of the MainForm, positioned below the main inventory controls for easy access during data entry.

1

Identify Your Recent Operations

QuickButtons display your recent transactions in order:

  • Button 1: Most recent transaction (top position)
  • Buttons 2-10: Previous transactions in chronological order
  • Button Text: Shows part number, operation, and location summary
2

Click to Auto-Fill

Single-click any QuickButton to automatically populate the inventory form:

  • Part Selection: Automatically selects the correct part number
  • Operation: Sets the operation (Machining, Assembly, etc.)
  • Location: Chooses the appropriate location
  • Quantity: Uses the same quantity from the original transaction
3

Modify and Save

Adjust any fields as needed and save:

  • Quantity Adjustment: Change quantity using keyboard shortcuts or direct entry
  • Location Override: Select different location if needed
  • Notes Addition: Add specific notes for this transaction
  • Save Operation: Complete the transaction normally (Ctrl+S)

üîÑ Button Management

Automatic Management

QuickButtons are automatically managed by the system. Manual reordering or deletion is not available - buttons update based on your actual transaction activity.

üìç Position Management

Button positions shift automatically with each new transaction:

  • New transactions move to position 1 (top)
  • Existing buttons shift down one position
  • Button 11+ are automatically removed
  • Position 10 becomes the "oldest" transaction

üßπ Automatic Cleanup

System maintains optimal performance through intelligent cleanup:

  • Only the last 10 transactions are kept
  • Duplicate transactions may be consolidated
  • Invalid or corrupted entries are automatically removed
  • Performance optimization runs during startup

üë§ User-Specific Storage

Each user maintains their own QuickButtons list:

  • Buttons are unique to each user account
  • No interference with other users' QuickButtons
  • Persistent across application sessions
  • Database synchronization ensures consistency

‚ùì Troubleshooting

QuickButtons are not appearing

Possible solutions:

  • Perform at least one successful inventory transaction to generate buttons
  • Check database connectivity - buttons require database access
  • Verify user privileges for accessing transaction history
  • Restart the application to refresh button loading

Buttons show outdated information

This typically indicates a database synchronization issue:

  • Wait a few seconds for background updates to complete
  • Try refreshing the main form (F5)
  • Check network connectivity to the database server
  • Contact IT support if problem persists

Clicking QuickButton doesn't fill form

Check these common issues:

  • Ensure the inventory form tab is active and visible
  • Verify that referenced parts/operations still exist in the database
  • Check for form validation errors preventing auto-fill
  • Try manually entering one field first to activate the form

Only showing fewer than 10 buttons

This is normal behavior if:

  • You have performed fewer than 10 total transactions
  • Some transactions were invalid or corrupted and removed
  • You're a new user with limited transaction history
  • Recent database cleanup removed older transaction data
