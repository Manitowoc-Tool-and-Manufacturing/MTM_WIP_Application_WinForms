üöÄ Basic Transfer Workflow

1

Search and Select Source Items

Use the search panel to locate items for transfer:

  • Part Search: Enter part number or description
  • Operation Filter: Select specific operation (Machining, Assembly, etc.)
  • Location Filter: Show items from specific locations
  • Show All: Display complete inventory for bulk operations
2

Select Items to Transfer

Choose items from the search results grid:

  • Single Selection: Click on one row for individual transfer
  • Multiple Selection: Hold Ctrl and click multiple rows
  • Row Highlighting: Selected rows show transfer status
3

Configure Transfer Details

Set up the transfer parameters in the transfer panel:

  • Destination Location: Select target location from dropdown
  • Quantity Control: Specify full or partial quantity transfer
  • Validation: System prevents same-location transfers
4

Execute Transfer

Complete the transfer operation:

  • Transfer Validation: System checks quantity limits and business rules
  • Confirmation Dialog: Review transfer details before execution
  • Transaction Log: Automatic audit trail creation

‚ùì Troubleshooting

Transfer button is disabled

Check that you have:

  • Selected at least one item in the left panel
  • Chosen a different destination location
  • Set a valid quantity (between 1 and available quantity)
  • Proper permissions for the destination location

"Cannot transfer to same location" error

The destination location cannot be the same as the current location. Select a different destination location from the dropdown.

Quantity exceeds available inventory

The transfer quantity cannot be more than what's currently available. Check the available quantity in the source location and adjust accordingly.

No items appear in search results

Try these solutions:

  • Use "Show All" to display complete inventory
  • Clear search filters and try again
  • Verify the part number or operation spelling
  • Check if items exist at the filtered location

Partial transfer creates duplicate entries

This is expected behavior. Partial transfers create a new inventory record at the destination while maintaining the remaining quantity at the source location.
